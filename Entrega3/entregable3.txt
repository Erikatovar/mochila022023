2. Que tipo de maquina virtual soporta virtual box?

               R: 	VirtualBox es una plataforma de virtualización que admite una amplia variedad de sistemas operativos invitados, incluyendo:

			    Microsoft Windows (Windows 3.1, Windows XP, Windows Vista, Windows 7, Windows 8, Windows 10, Windows Server 2003, Windows Server 2008, Windows Server 2012, Windows Server 2016, etc.)
			    Linux (Ubuntu, Debian, Fedora, Red Hat Enterprise Linux, CentOS, etc.)
			    macOS (anteriormente conocido como OS X)
			    Solaris (OpenSolaris, Solaris 10, Solaris 11)
    BSD (FreeBSD, OpenBSD, NetBSD)
	
Además, VirtualBox también puede ejecutar otros sistemas operativos invitados como DOS, OS/2, y otros sistemas operativos menos conocidos. VirtualBox es una plataforma de virtualización muy versátil y puede funcionar con muchos sistemas operativos diferentes.

4.Si tengo mas de una maquina virtual instalada, y una se rompe,  esto afecta a las demas?  por que?

	R:En principio, si tienes varias máquinas virtuales instaladas en tu ordenador, una máquina virtual que se rompa no debería afectar a las demás directamente. Cada máquina virtual es un entorno aislado que funciona de forma independiente del resto de las máquinas virtuales y del sistema operativo anfitrión.

	Sin embargo, hay algunas situaciones en las que una máquina virtual que se rompe podría afectar a otras máquinas virtuales en tu sistema. Por ejemplo, si la máquina virtual que se rompe consume una gran cantidad de recursos del sistema, como la memoria RAM o el procesador, esto podría ralentizar el rendimiento de otras máquinas virtuales que estén ejecutándose en el mismo sistema. Además, si la máquina virtual que se rompe comparte archivos o recursos con otras máquinas virtuales, podría haber un efecto de cascada que afecte a otras máquinas virtuales.

	En resumen, aunque una máquina virtual que se rompe en principio no debería afectar a las demás, es importante tener en cuenta que el rendimiento de las otras máquinas virtuales podría verse afectado indirectamente si la máquina virtual que se rompe consume muchos recursos del sistema o comparte recursos con otras máquinas virtuales.
